## ⏳ 2. Telegram недоступен с VPS (RU-блок) — нужно решение по прокси

t.me и api.telegram.org с VPS не достучаться (`Network unreachable`). Значит на VPS:
- лента «Из каналов» пустая (посты не скрейпятся);
- ТГ-события не извлекаются;
- пятничный бот-дайджест (GEO-92) не отправит.

**План:** поднять прокси вне РФ и пустить через него ТОЛЬКО ТГ-трафик (не весь VPS) —
например, контейнер пайплайна/бота через `network_mode: service:warp` (Cloudflare
WARP, бесплатный) или WireGuard на бесплатный внешний узел. Риск: WARP-эндпоинты
из РФ иногда тоже режут — нужно проверить на месте. Официальные MTProto-прокси
Telegram не подходят (это для клиентов, не для Bot API / web-скрейпа).

**Что подтвердить владельцу:** ок ли пробовать бесплатный Cloudflare WARP первым;
если не заведётся из РФ — нужен дешёвый внешний relay (мелкий VPS/VPN вне РФ).
