## 1. Секреты для VPS-пайплайна (.env) — нужен ключ

**Что:** на VPS `/root/repos/msk-go/.env` создан, но пустой. Пайплайн работает
деградированно: KudaGo (бескейный) + curated. Не работают:
- **OPENROUTER_KEY** → LLM-извлечение событий из Telegram-постов отключено.
- **TIMEPAD_TOKEN** → источник «Спорт» (Timepad) пропускается.

**Почему агент сам не взял:** ключи лежат только в GitHub Actions Secrets (локально
недоступны). Сканирование кред-сторов на VPS (openclaw.json / .bashrc) заблокировано
классификатором безопасности — намеренно не обходил.

**Что сделать владельцу:** вписать значения в `/root/repos/msk-go/.env`:
```
OPENROUTER_KEY=...
TIMEPAD_TOKEN=...
```
Примечание: в конфиге OpenClaw (`/root/.openclaw/openclaw.json`) уже есть рабочий
OpenRouter-ключ — его можно переиспользовать.

## 2. ⚠️ Telegram недоступен с VPS (RU-блок) — архитектурное решение

**Что обнаружено:** из VPS `t.me/s/<channel>` недоступен (`Network is unreachable`,
errno 101) — Telegram заблокирован для российских серверов (подтверждается memory
про OpenClaw: Telegram с VPS заблокирован). Значит **скрейп ТГ-каналов с VPS не
работает в принципе**, даже с OPENROUTER_KEY (LLM есть, но постов нет).

**Следствие:** если пайплайн полностью переедет на VPS (отказ от GitHub Actions),
источник Telegram умрёт. Сейчас не критично: GitHub Actions (US-раннеры) ТГ берёт
нормально и остаётся прод-источником.

**Что сделать владельцу (решение):** один из:
- (a) оставить сбор ТГ на GitHub Actions, VPS-пайплайн — для остальных источников;
- (b) поднять на VPS прокси вне РФ для t.me (как proxy6/proxychains у OpenClaw) и
  гонять ТГ через него;
- (c) согласиться, что VPS-пайплайн без ТГ (только KudaGo/Timepad/curated).

Также пятничный TG-дайджест-бот (GEO-92) с VPS упрётся в этот же блок → нужен прокси.

## 3. Paperclip-оркестратор (GEO-101) — нужен ANTHROPIC_API_KEY + аккаунт

**Что:** установка/запуск Paperclip и самого агента-куратора заблокированы.

**Почему:**
- Paperclip (`npx paperclipai onboard`) и агент-рантайм требуют
  **ANTHROPIC_API_KEY с usage-credits** (BLOCKING-пререквизит, см.
  `vault/projects/aim/{AIM} {task} Пилот Paperclip + gstack`). claude.ai-логин/Max
  не аутентифицируют программный запуск.
- Запуск `npx paperclipai` на общем VPS заблокирован классификатором безопасности.
- bun (харнесс gstack) на VPS не установлен.

**Сделано без ключа:** каркас агента-куратора готов и закоммичен —
`agents/curator.md` (роль/контракт/guardrails) + `proposals/` (конвенция).
Запустится, как только появится ключ.

**Что сделать владельцу:**
1. Завести API-ключ (platform.claude.com) с небольшим usage-credit ($20–50).
2. Решить хостинг Paperclip (рядом с ozon-analytics на VPS vs локально) — VPS уже
   нагружен (Airflow + Postgres + ozon-analytics + delta-panel, ~3 ГБ RAM занято
   из 8).
3. Дать добро на `npx paperclipai onboard` (или выполнить самому: `! ssh ...`).

## 4. Перенос фронта на VPS → приватный репозиторий (GEO-103) — решение + домен

**Жёсткий факт:** приватный репозиторий ЛОМАЕТ GitHub Pages на free-тарифе.
Поэтому **репо приватным НЕ делал** — прод (https://georgypasyuk.github.io/msk-go/)
должен жить, пока владельца нет.

**Сделано:** статичный фронт поднят на VPS как контейнер-зеркало (nginx) на
свободном порту — доказательство, что фронт работает с VPS на данных VPS-пайплайна
(см. `DEPLOY.md`, сервис `web`). DNS/домен и TLS не трогал.

**Что сделать владельцу (решения):**
1. **Домен:** под каким адресом отдавать фронт с VPS? (напр.
   `msk.pasyukgeorgy.fvds.ru` или путь на существующем nginx). Нужно решение +
   возможно DNS-запись.
2. После того как фронт на VPS реально отдаётся по домену с TLS — **только тогда**
   можно делать репозиторий приватным (или забрать данные из git и оставить Pages).
3. Ключ Яндекс-карт (GEO-103 [SEC]): ограничить по referer в кабинете Яндекса
   (агент не имеет доступа к кабинету) — отдельная задача.
