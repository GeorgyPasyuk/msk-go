# Агент-куратор msk-go (GEO-101)

Один узкий агент. Не рой. Читает канон + данные, **пишет предложения**, ничего
не применяет. Гейт: предложения уходят как PR/файл в `proposals/`, решение —
за человеком/CI (GEO-100).

## Контекст (вход)

- **Канон (источник истины):** `vault/projects/msk-go/PROJECT.md` (на VPS:
  `/root/vault/projects/msk-go/PROJECT.md`, если синкается; иначе передать текст
  канона в промпт).
- **Данные:** `runtime/data/events.json`, `runtime/data/telegram.json`
  (на VPS после прогона пайплайна).
- **Код:** `scripts/`, `assets/` — только чтение для понимания.

## Что делает (узкая роль: QA качества данных и подбора)

Каждый запуск (по cron, напр. раз в день после пайплайна):

1. Прочитать канон → понять scope/anti-scope и вкус-профиль владельца.
2. Прочитать свежий `events.json` (+ past) и `telegram.json`.
3. Найти проблемы и возможности, например:
   - события из **anti-scope** просочились (филармония/театр/детское/реклама) →
     предложить правило фильтра;
   - подозрения на **дубли**, которые не схлопнул кросс-дедуп (GEO-94);
   - **протухание**: `generated_at` старый, источник отвалился, пустые категории;
   - кандидаты в `curated.json` (сезонное/заголовочное, чего нет в API);
   - заметные пробелы по вкус-профилю (мало вело/сап/опен-эйров и т.п.).
4. Записать находки в `proposals/` (см. формат ниже). **Код и данные не трогать.**

## Выход (контракт)

Один markdown-файл на запуск: `proposals/YYYY-MM-DD-<краткий-слаг>.md`.

```markdown
# Предложения куратора — YYYY-MM-DD

## Сводка
<1–3 строки: что проверено, главное>

## Находки
- [тип: anti-scope|дубль|свежесть|пробел|curated] <описание> · доказательство (id/название/дата) · рекомендация

## Предлагаемые изменения (НЕ применены)
- файл `scripts/fetch_events.py`: <что и зачем> (только текстом, не патчем в прод)
- curated-кандидаты: <список объектов для ручной проверки>
```

## Жёсткие правила (guardrails)

- **Только предложения.** Никаких правок кода/данных/прод, никаких коммитов в `main`.
- Слово «движ» не употреблять (термин-запрет проекта).
- Не плодить файлы: один файл предложений на запуск, в `proposals/`.
- Не выдумывать события — опираться только на то, что в данных/каноне.

## Харнесс

Запуск как агент Paperclip (оркестрация) или вручную через gstack-скиллы
(`/review` для разбора, анализ канона+данных). Требует `ANTHROPIC_API_KEY` —
см. `ESCALATIONS.md` (на момент создания каркаса ключа на VPS нет → агент не
запущен, каркас готов к старту).
